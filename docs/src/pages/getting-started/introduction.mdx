# Introduction

Latty is a **framework-agnostic design system** built on two primitives: design tokens and Web Components.

## How it works

Everything flows top-down from a single config file:

```
tokens.config.json          ← base brand colors
       ↓
@latty/tokens               ← generates palettes, spacing, typography
       ↓
  tokens.css                ← CSS custom properties (--lt-color-primary-500, etc.)
  tokens.json               ← for tooling and documentation
  tokens.js                 ← for JS/TS consumers
       ↓
@latty/web                  ← Lit Web Components that consume CSS vars
       ↓
@latty/react / @latty/angular  ← thin framework wrappers (optional)
```

Components never hardcode colors, sizes, or spacing — they only reference `--lt-*` custom properties. This means a single token change ripples through every component automatically.

## Package structure

| Package | What it does |
|---|---|
| `@latty/tokens` | Generates CSS vars, JSON, and JS from brand config |
| `@latty/web` | 16 Lit Web Components with `lt-` prefix |
| `@latty/icons` | Icon system backed by Iconoir (800+ icons), pluggable |
| `@latty/react` | React wrappers around `@latty/web` |
| `@latty/angular` | Angular wrappers around `@latty/web` |

## Naming conventions

All public APIs follow a consistent prefix scheme:

- **npm scope** — `@latty/*`
- **Custom element names** — `lt-button`, `lt-textfield`, `lt-dialog`, …
- **CSS custom properties** — `--lt-color-*`, `--lt-spacing-*`, `--lt-border-radius`, …

## Why Web Components?

Web Components run natively in every browser without a framework runtime. This means:

- **No adapter needed** for vanilla HTML, Vue, Svelte, or any future framework
- **Encapsulation via Shadow DOM** — styles can't bleed in or out accidentally
- **Reactive attributes** — properties reflect to HTML attributes, so server-rendered markup works

React and Angular wrappers exist for cases where you want typed props and idiomatic event binding in those frameworks.
