# Design Tokens

Design tokens are the single source of truth for every visual decision in Latty. They are generated at build time from `tokens.config.json` and shipped as three artifacts:

| Output | Path | Use case |
|---|---|---|
| CSS custom properties | `@latty/tokens/dist/tokens.css` | Any browser context |
| JSON | `@latty/tokens/dist/tokens.json` | Tooling, documentation |
| JavaScript module | `@latty/tokens/dist/tokens.js` | JS/TS consumers |

## Token categories

| Category | CSS prefix | Example |
|---|---|---|
| Colors | `--lt-color-*` | `--lt-color-primary-500` |
| Spacing | `--lt-spacing-*` | `--lt-spacing-4` |
| Border radius | `--lt-border-radius` | `0.5rem` |
| Elevation | `--lt-elevation-*` | `--lt-elevation-2` |
| Typography | `--lt-font-family` | `"Asap", sans-serif` |

## Import

```css
@import '@latty/tokens/dist/tokens.css';
```

```js
import '@latty/tokens/dist/tokens.css';       // CSS vars in browser
import tokens from '@latty/tokens/dist/tokens.js'; // JS object
```

## Customization

Override any token on `:root` after the import to theme the system:

```css
@import '@latty/tokens/dist/tokens.css';

:root {
  --lt-border-radius: 2px;
  --lt-font-family: 'Inter', sans-serif;
}
```

For brand-level color changes, edit `tokens.config.json` and rebuild:

```bash
pnpm --filter @latty/tokens build
```
